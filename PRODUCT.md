# Product

## Register

product

## Users
Internal operators and technical staff who need to upload a small document set, search it, and ask grounded questions against source files. They use the product during focused work and need fast confirmation, clear status, and traceable answers.

## Product Purpose
This product is an internal RAG file manager for storing documents, parsing them into searchable chunks, retrieving relevant source passages, and answering questions with citations. Success means a user can upload a document, see processing status, search across allowed files, ask a question, and verify the answer from page-level citations.

## Brand Personality
Reliable, restrained, precise. The interface should feel like a calm operations tool: useful first, polished enough to trust, and never theatrical.

## Anti-references
Avoid marketing landing-page treatment, oversized hero sections, decorative AI gradients, glassy panels, chat apps that hide sources, and generic card grids. The product must not look like a public SaaS homepage.

## Design Principles
- Make retrieval inspectable: every answer should carry source evidence.
- Keep workflow visible: upload, processing, ready, failed, search, and chat states should be easy to scan.
- Prefer quiet density: this is a workbench for repeated use, not a brochure.
- Make failures actionable: parse and API errors should be short, plain, and tied to the affected document or query.
- Protect internal content: permissions and session behavior should be visible through consistent affordances, not decorative warnings.

## Accessibility & Inclusion
Target WCAG AA contrast, keyboard-usable forms and buttons, visible focus states, reduced-motion-safe transitions, readable Vietnamese and English text, and non-color-only status indicators.
